# 피보나치 수열 - 순환적 방법 vs 재귀적 방법

## 구현

### 순환적 방법

```c
long long fib_iterative(int n) {
    if (n <= 2) return 1;
    long long a = 1, b = 1, c;
    for (int i = 3; i <= n; i++) {
        c = a + b;
        a = b;
        b = c;
    }
    return b;
}
```

### 재귀적 방법

```c
long long fib_recursive(int n) {
    if (n <= 2) return 1;
    return fib_recursive(n - 1) + fib_recursive(n - 2);
}
```

## 분석

### 시간 복잡도

순환적 방법은 반복문을 N번 돌기 때문에 **O(N)** 이다. 변수 세 개만 쓰므로 공간도 **O(1)** 로 일정하다.

재귀적 방법은 `fib(n) = fib(n-1) + fib(n-2)` 를 그대로 호출로 표현한다. 문제는 이미 계산한 값을 저장하지 않고 매번 다시 계산한다는 점이다. 예를 들어 `fib(6)`을 구할 때 `fib(3)`은 3번, `fib(2)`는 5번 호출된다. N이 1 커질수록 호출 횟수가 약 2배씩 늘어나서 **O(2^N)** 이 된다.

### 측정 결과로 보면

순환적 방법은 N=45까지 측정값이 거의 0에 수렴했다. 워낙 빠르다 보니 `clock_gettime`의 분해능(1,000ns) 안에 들어오는 경우가 대부분이었다.

재귀적 방법은 N=20부터 슬슬 느려지기 시작해서, N=45에서는 약 **3.3초**가 걸렸다. N=40 대비 N=45가 약 10배 느린데, 이론상 2^5 = 32배여야 하지만 캐시나 분기 예측 등의 영향으로 실제 배율은 다소 다르게 나온다.

### 정리

| 항목          | 순환적 | 재귀적    |
| ------------- | ------ | --------- |
| 시간 복잡도   | O(N)   | O(2^N)    |
| 공간 복잡도   | O(1)   | O(N)      |
| N=45 수행시간 | ~0 ms  | ~3,334 ms |

코드 자체는 재귀가 수학 정의와 똑같이 생겨서 읽기 편하지만, 성능 면에서는 순환적 방법과 비교가 안 된다.
