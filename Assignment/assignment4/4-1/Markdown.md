# 과제 4-1: GCD(최대공약수) 시간복잡도와 Big-O

## 1. 제출한 GCD 알고리즘

- **알고리즘 방식**: 유클리드 호제법 (반복문)

```c
int gcd(int a,int b) {
    if(a < 0) a = -a;
    if(b < 0) b = -b;

    if(b == 0) return a;

    while(b != 0) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}
```

---

## 2. 알고리즘 동작 원리

유클리드 호제법은 다음 성질을 반복 적용한다.

```
GCD(a,b) = GCD(b, a mod b) (단, b != 0)
GCD(a,0) = a
```

매 반복마다 b가 a mod b로 교체되므로, 나머지가 0이 될 때까지 반복한다.

---

## 3. 시간복잡도란?

- 시간복잡도는 입력값이 커질수록 알고리즘의 실행 시간이 어떻게 변하는지를 나타내는 개념이다.
  이를 통해 알고리즘이 얼마나 효율적인지 판단할 수 있다.

### GCD 알고리즘의 시간복잡도 계산

### 동작 원리

`while` 루프를 반복할 때마다 `b`가 `a % b`(나머지)로 바뀐다.  
나머지는 항상 `b`보다 작으므로, 반복할수록 숫자가 줄어들어 결국 0이 된다.

### 예시: GCD(21, 13)

| 단계 |  a  |  b  |  a % b   |
| :--: | :-: | :-: | :------: |
|  1   | 21  | 13  |    8     |
|  2   | 13  |  8  |    5     |
|  3   |  8  |  5  |    3     |
|  4   |  5  |  3  |    2     |
|  5   |  3  |  2  |    1     |
|  6   |  2  |  1  | 0 → 종료 |

총 6번 만에 끝났다.

### 왜 O(log N)인가?

유클리드 호제법은 반복할 때마다 b의 값이 a % b로 바뀌면서 점점 작아진다.  
이 과정에서 값은 빠르게 감소하며, 결국 0이 되어 반복이 종료된다.

특히 최악의 경우에도 반복 횟수는 입력값의 크기에 대해 로그(log) 수준으로 증가한다.

따라서 while 루프의 반복 횟수는 O(log N)이라고 할 수 있다.

### 실제로 확인해보면

|  입력 N   | 반복 횟수 (최대) |
| :-------: | :--------------: |
|    10     |      약 4번      |
|    100    |      약 7번      |
|   1,000   |     약 10번      |
| 1,000,000 |     약 20번      |

N이 커지더라도 반복 횟수는 많이 늘어나지 않고, 조금씩만 증가하는 것을 확인할 수 있다.

---

## 4. Big-O 표기법이란?

- Big-O는 시간복잡도를 표현하는 방법이다. 입력 크기를 N이라 할 때, 최악의 경우 몇 번 연산하는지를 나타낸다.

|   표기   | 의미                       | 예시                        |
| :------: | :------------------------- | :-------------------------- |
|   O(1)   | N이 아무리 커도 항상 1번   | 배열에서 인덱스로 값 꺼내기 |
|   O(N)   | N이 커지면 그만큼 느려짐   | 배열을 처음부터 끝까지 탐색 |
| O(log N) | N이 커져도 조금씩만 느려짐 | 이진 탐색, 유클리드 GCD     |
|  O(N²)   | N이 2배 커지면 4배 느려짐  | 버블 정렬                   |

## 5. Big-O 정리

| 구분           | 복잡도       |
| :------------- | :----------- |
| **시간복잡도** | **O(log N)** |
| 공간복잡도     | O(1)         |

- 시간복잡도에서 N은 입력값 a와 b 중에서 더 큰 값을 의미한다.
- 시간복잡도 O(log N)은 입력값이 커지더라도 반복 횟수가 크게 증가하지 않는다는 뜻이다.
- 공간복잡도 O(1)이란, N이 아무리 커도 메모리 사용량이 변하지 않는다는 뜻이다.
