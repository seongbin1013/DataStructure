# 과제 4-1: GCD(최대공약수) 시간복잡도와 Big-O

## 1. 제출한 GCD 알고리즘

- **파일 링크**: [main.c](./main.c)
- **알고리즘 방식**: 유클리드 호제법 (반복문)

```c
int gcd(int a, int b) {
    if(a < 0) a = -a;
    if(b < 0) b = -b;

    while(b != 0) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}
```

---

## 2. 알고리즘 동작 원리

유클리드 호제법은 다음 성질을 반복 적용한다.

```
GCD(a,b) = GCD(b, a mod b) (단, b != 0)
GCD(a,0) = a
```

매 반복마다 b가 a mod b로 교체되므로, 나머지가 0이 될 떄가지 반복하다.

### 예시: GCD(21,13)

| 단계 |  a  |  b  | a mod b  |
| :--: | :-: | :-: | :------: |
|  1   | 21  | 13  |    8     |
|  2   | 13  |  8  |    5     |
|  3   |  8  |  5  |    3     |
|  4   |  5  |  3  |    2     |
|  5   |  3  |  2  |    1     |
|  6   |  2  |  1  | 0 → 종료 |

---

## 3. 시간복잡도란?

- 알고리즘이 입력이 커질수록 얼마나 느려지는지를 나타내는 척도이다.

### 시간복잡도 계산

### 핵심 관찰: 두 단계마다 값이 절반 이하로 줄어든다

`a = 21, b = 13`이라 하면, 두 번 반복 후 `b`는 5가 된다.  
21의 절반(10.5)보다 작다. 이처럼 **2번 반복할 때마다 값이 반드시 절반 이하**가 된다.

값이 N에서 시작해서 절반씩 줄어들며 1에 도달하려면:

```
N → N/2 → N/4 → ... → 1
```

이 과정은 약 log₂(N)번 반복된다.  
따라서 `while` 루프의 총 반복 횟수는:

```
O(log N)
```

### 예시로 확인

|  입력 N   | 대략적인 반복 횟수 |
| :-------: | :----------------: |
|    10     |        ~4번        |
|    100    |        ~7번        |
|   1,000   |       ~10번        |
| 1,000,000 |       ~20번        |

N이 100배 커져도 반복 횟수는 3번밖에 늘지 않는다. 매우 빠른 알고리즘이다.

---

## 4. Big-O 표기법이란?

- Big-O는 시간복잡도를 표현하는 방법이다. 입력 크기를 N이라 할 때, 최악의 경우 몇 번 연산하는지를 나타낸다.

|   표기   | 의미                       | 예시                        |
| :------: | :------------------------- | :-------------------------- |
|   O(1)   | N이 아무리 커도 항상 1번   | 배열에서 인덱스로 값 꺼내기 |
|   O(N)   | N이 커지면 그만큼 느려짐   | 배열을 처음부터 끝까지 탐색 |
| O(log N) | N이 커져도 조금씩만 느려짐 | 이진 탐색, 유클리드 GCD     |
|  O(N²)   | N이 2배 커지면 4배 느려짐  | 버블 정렬                   |

### Big-O 정리

| 구분           | 복잡도       |
| :------------- | :----------- |
| **시간복잡도** | **O(log N)** |
| 공간복잡도     | O(1)         |

- `N = max(a, b)`
- `while` 루프 내부 연산(대입, 나머지 연산)은 모두 O(1)
- 추가 메모리는 `temp` 변수 하나뿐 → O(1)

---

## 5. 알고리즘 방식 비교

| 방식                 | 시간복잡도 | 공간복잡도 | 비고                    |
| :------------------- | :--------: | :--------: | :---------------------- |
| **반복문 (본 구현)** |  O(log N)  |  **O(1)**  | 스택 오버플로우 없음    |
| 재귀                 |  O(log N)  |  O(log N)  | 호출 스택 사용          |
| 뺄셈 기반            |    O(N)    |    O(1)    | 입력이 클수록 매우 느림 |

본 구현은 반복문 방식이므로 재귀 방식보다 **공간 효율이 더 좋다**.
